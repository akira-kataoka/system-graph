# System Graph 🕸️

社内システム・テーブル・データ連携を **Obsidian 風のノードグラフ**で可視化・管理する単一ファイルの Web アプリです。外部ライブラリ・バックエンド不要で、GitHub Pages にそのまま置くだけで動きます。

## 特長

企業のシステム構成（データリネージ）を、**システム › データベース › テーブル** と **連携基盤 › ジョブ** の入れ子構造で可視化します。

- **入れ子コンテナ**：どのシステムの・どのDB（エンジン付き）の・どのテーブルか、どのETL基盤のどのジョブかが一目で分かる
- **色分けモード**：`種別` / `ステータス（稼働中・更改中・廃止予定…）` / `ヘルス（正常・注意・障害）` を切替
- **障害・更改の警告バッジ**：未解決インシデントや更改予定が近いシステムに常時 `!` を表示（全体像の中で問題が即座に目立つ）
- **概要パネル**：システム数などのKPI、稼働ステータス内訳、**要対応リスト**（障害・直近の更改）をワンクリックで確認
- **影響分析**：ノード選択 →「下流の影響範囲 / 上流の依存元」で、更改・障害の波及先をハイライト
- **登録項目**：名前・種別・所属・説明・所有者・技術・タグに加え、稼働ステータス／重要度／環境・ホスティング／ベンダー／導入日・更改予定日、**インシデント一覧**、テーブルの**カラム定義**
- **データの流れ**：粒子が線上を流れ、**読取＝青 / 書込＝緑 / 参照＝紫** で色分け・グロー表示。**●の大きさは連携データ量**に連動（向き・種類・規模が一目瞭然）
- **データ連携の詳細**：線（連携）をクリックすると、**抽出条件（WHERE）・連携方式（全件/差分/CDC/UPSERT）・連携間隔・実行時間帯・データ量/規模・増分キー・変換ロジック・項目マッピング（元→先・加工ルール）** を編集
- **箱（コンテナ）はテーブルが無くてもドラッグで移動可能**
- **テーブル**は階層の終端として ● で表現。**カラム定義**（列名・型・PK）を登録
- **1システムに複数データベース**、管理体制（**オーナー・管理部署・運用保守**）を登録
- **台帳(Markdown)出力**：システム構成・カラム・抽出条件・項目マッピングを含むドキュメントを書き出し
- ホバーで要点ツールチップ・接続数連動のノードサイズ・ミニマップ
- **ドラッグ**：ノードもコンテナ枠（配下ごと）も移動でき、ドロップ位置に固定（ダブルクリックで解除／「整列」で全解除）
- **検索**・**localStorage 自動保存** + **JSON エクスポート/インポート**・ライト/ダークテーマ・絵文字なしのSVGアイコンUI

## 使い方

| 操作 | 方法 |
|---|---|
| ノード追加 | 上部の「システム / テーブル / 連携 / 外部」ボタン |
| リンク作成 | 「🔗 リンク作成」→ 元ノード → 対象ノードの順にクリック（`L`キー） |
| 編集 | ノードをクリック → 右パネルで編集（自動保存） |
| 削除 | ノード選択 → 🗑 削除（`Delete`キー） |
| 移動 | ノードをドラッグ / 背景ドラッグでパン / ホイールでズーム |
| 全体表示 | 「⊡」ボタン（`F`キー） |
| 再整列 | 「🎯 整列」 |
| 保存・共有 | 「⬇ JSON」で書き出し → リポジトリにコミット、「⬆ 読込」で復元 |

## データの共有方法

ブラウザの localStorage に自動保存されるため、同じ端末・ブラウザなら再訪時に復元されます。
チームで共有したい場合は **⬇ JSON でエクスポート → `system-graph.json` をこのリポジトリにコミット**し、
各自が **⬆ 読込**で取り込む運用がおすすめです。

## GitHub Pages で公開する

```bash
cd system-graph
git init
git add .
git commit -m "Add system graph"
gh repo create system-graph --public --source=. --push
gh api -X POST repos/akira-kataoka/system-graph/pages -f "source[branch]=main" -f "source[path]=/"
```

数分後、`https://akira-kataoka.github.io/system-graph/` で公開されます。
（既存リポジトリの一部として置く場合は、Settings → Pages でソースブランチ／フォルダを指定してください。）
