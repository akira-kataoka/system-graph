# System Graph 🕸️

社内システム・テーブル・データ連携を **Obsidian 風のノードグラフ**で可視化・管理する単一ファイルの Web アプリです。外部ライブラリ・バックエンド不要で、GitHub Pages にそのまま置くだけで動きます。

## 特長

- **力学レイアウトのグラフ表示**（自動整列 / ドラッグ / ズーム・パン）
- **4種のノード**: システム（●青）/ テーブル（■緑）/ データ連携・ジョブ（◆橙）/ 外部サービス（●桃）
- **有向リンク**でデータの流れ・参照関係を表現（ラベル付き）
- ノードをクリックで**インスペクタ**編集（名前・種別・説明・所有者・技術スタック・タグ・接続一覧）
- **検索**でノードを絞り込み・フォーカス
- **localStorage 自動保存** + **JSON エクスポート/インポート**（チーム共有・バックアップ用）
- ライト/ダークテーマ対応

## 使い方

| 操作 | 方法 |
|---|---|
| ノード追加 | 上部の「システム / テーブル / 連携 / 外部」ボタン |
| リンク作成 | 「🔗 リンク作成」→ 元ノード → 対象ノードの順にクリック（`L`キー） |
| 編集 | ノードをクリック → 右パネルで編集（自動保存） |
| 削除 | ノード選択 → 🗑 削除（`Delete`キー） |
| 移動 | ノードをドラッグ / 背景ドラッグでパン / ホイールでズーム |
| 全体表示 | 「⊡」ボタン（`F`キー） |
| 再整列 | 「🎯 整列」 |
| 保存・共有 | 「⬇ JSON」で書き出し → リポジトリにコミット、「⬆ 読込」で復元 |

## データの共有方法

ブラウザの localStorage に自動保存されるため、同じ端末・ブラウザなら再訪時に復元されます。
チームで共有したい場合は **⬇ JSON でエクスポート → `system-graph.json` をこのリポジトリにコミット**し、
各自が **⬆ 読込**で取り込む運用がおすすめです。

## GitHub Pages で公開する

```bash
cd system-graph
git init
git add .
git commit -m "Add system graph"
gh repo create system-graph --public --source=. --push
gh api -X POST repos/akira-kataoka/system-graph/pages -f "source[branch]=main" -f "source[path]=/"
```

数分後、`https://akira-kataoka.github.io/system-graph/` で公開されます。
（既存リポジトリの一部として置く場合は、Settings → Pages でソースブランチ／フォルダを指定してください。）
